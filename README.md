## Як користуватись

1. **Оберіть інструмент** — автоматично підставляться специфікації і приклади цін
2. **Введіть макс. збиток** — скільки USD ви готові втратити на цій угоді (рекомендовано ≤2% балансу)
3. **Введіть стоп-лос в пунктах** — відстань від ціни входу до стоп-лосу
4. **Введіть ціну входу** — реальна ціна відкриття позиції
5. **Введіть тейк-профіт** — цільова ціна закриття
6. **Оберіть напрямок** — Buy (Long) або Sell (Short)
7. **Введіть баланс** — для розрахунку % ризику

Всі результати оновлюються миттєво.

## Логіка розрахунку

### Основна формула

```
Збиток на 1 лот = SL пунктів × вартість пункту ($)
Об'єм (лоти)   = Макс. збиток / Збиток на 1 лот
Профіт         = Об'єм × TP пунктів × вартість пункту
R:R            = Профіт / Макс. збиток
```

### Що рахує калькулятор

| Результат | Формула | Опис |
|---|---|---|
| **Об'єм (лоти)** | `Макс.збиток / (SL піп × ціна піпа)` | Дробове: 0.67, 1.25, 10.0 |
| **Макс. збиток** | `Об'єм × SL піп × ціна піпа` | В доларах USD |
| **Потенц. профіт** | `Об'єм × TP піп × ціна піпа` | В доларах USD |
| **R:R** | `Профіт / Збиток` | Відношення прибуток/ризик |
| **Ціна SL** | Buy: `Entry − SL×pipSize` | Точна ціна для MT4 |
| **% від балансу** | `Збиток / Баланс × 100` | Ризик на угоду |

### Вибір ціни входу (логіка KovachLog)

При роботі з сигналами Ковача використовується зона входу (entry_low – entry_high). Реалістична ціна входу:
- **ЛОНГ (Buy)** → вхід по **верху зони** (entry_high) — ціна може не дійти до низу зони
- **ШОРТ (Sell)** → вхід по **низу зони** (entry_low) — ціна може відскочити від верху

Саме від цієї ціни рахується SL в пунктах для введення в калькулятор.

**Після реального входу** — перерахуй з фактичною ціною входу (entryFact).

## Зв'язок з KovachLog

Колонки **Профіт $**, **Луз $**, **R:R** в таблиці KovachLog розраховуються за тими самими AMarkets специфікаціями що і цей калькулятор.

**При імпорті сигналів:**
- ЛОНГ → R:R рахується від entry_high
- ШОРТ → R:R рахується від entry_low
- Луз $ = завжди $500 (константа max_loss)

**Після входу в зону:**
- При фіксації реальної ціни входу (entryFact) — Профіт$ і R:R автоматично перераховуються

Тому R:R в таблиці = R:R в калькуляторі при однакових параметрах.

## Приклади розрахунків

### EURUSD — Buy, сигнал Ковача
```
Зона входу: 1.1390–1.1405
Вхід (верх зони): 1.1405
Stop Loss: 1.135
Take Profit: 1.150

SL пунктів: |1.1405 − 1.135| / 0.0001 = 55 пунктів
TP пунктів: |1.150  − 1.1405| / 0.0001 = 95 пунктів

Збиток на 1 лот = 55 × $10 = $550
Об'єм = $500 / $550 = 0.91 лота
Профіт = 0.91 × 95 × $10 = $864
R:R = $864 / $500 = 1.73
```

### WTI (MCL) — Buy, сигнал Ковача
```
Зона входу: 68.5–70.0
Вхід (верх зони): 70.0
Stop Loss: 68.0
Take Profit: 85.0

SL пунктів: |70.0 − 68.0| / 0.01 = 200 пунктів
TP пунктів: |85.0 − 70.0| / 0.01 = 1500 пунктів

Збиток на 1 лот = 200 × $10 = $2,000
Об'єм = $500 / $2,000 = 0.25 лота
Профіт = 0.25 × 1500 × $10 = $3,750
R:R = $3,750 / $500 = 7.5
```

### USDCAD — Sell, сигнал Ковача
```
Зона входу: 1.4175–1.4200
Вхід (низ зони для SELL): 1.4175
Stop Loss: 1.421
Take Profit: 1.400

SL пунктів: |1.4175 − 1.421| / 0.0001 = 35 пунктів
TP пунктів: |1.400  − 1.4175| / 0.0001 = 175 пунктів

Збиток на 1 лот = 35 × $7.05 = $246.75
Об'єм = $500 / $246.75 = 2.03 лота
Профіт = 2.03 × 175 × $7.05 = $2,503
R:R = $2,503 / $500 = 5.0
```

### XAUUSDb (1OZ) — Buy, сигнал Ковача
```
Зона входу: 3900–3950
Вхід (верх зони): 3950
Stop Loss: 3800
Take Profit: 4250

SL пунктів: |3950 − 3800| / 0.01 = 15,000 пунктів
TP пунктів: |4250 − 3950| / 0.01 = 30,000 пунктів

Збиток на 1 лот = 15,000 × $0.10 = $1,500
Об'єм = $500 / $1,500 = 0.33 лота
Профіт = 0.33 × 30,000 × $0.10 = $990
R:R = $990 / $500 ≈ 2.0
```

## Ризик-менеджмент

| Ризик на угоду | Оцінка |
|---|---|
| ≤ 1% балансу | ✅ Консервативно |
| 1–2% балансу | ✅ Рекомендовано |
| 2–5% балансу | ⚠️ Підвищений ризик |
| > 5% балансу | 🔴 Небезпечно |

**При балансі $600:** max_loss ≤ $12 (2%). Але при такому малому балансі навіть 0.01 лота дає відчутний ризик — рекомендовано починати з мінімальних об'ємів.

**Одночасні позиції:** якщо відкрито 3 угоди по $500 — сумарний ризик $1,500. При балансі $600 це неможливо. Тому при малому балансі: max_loss = $50–100 на угоду.
